No more
Unit trouble in
Supermarkets

With our app, you will no longer be frustrated with
“lb” unit during your shopping.

A new place, different culture

Traveling, studying, working in different region. You might be facing a lot of unfamiliar norms and cultures.

Shopping is also different

Different culture also exists when shopping, like in supermarket. When you try to buy fruit or vegetables, you could see “lb” unit.

What is lb, How much will I pay?

By using “lb” unit, It is very difficult to grasp how much you`ll have to pay after all. Do you want to know how much it will cost if you buy only one apple ?

Survey

We conducted a survey to collect users' opinions. By asking several questions about supermarket unit and what they think about the “lb” unit, we were able to understand their desire.

Q: Have you ever felt any frustration, about the "lb" unit, when buying something?

More than 70% of the users experienced some frustration when they face to “lb” unit in the supermarket.

Q: Do you want to know how much it costs, if you buy one piece of food? For example, a price for one apple.

More than 90% of the users desire to know how much it will cost for each piece of food when they buy food.

Q: When you buy some fruits or vegetables, do you use a scale?

Even though, around 70% of the users don’t use scale in the supermarket when they buy fruit or vegetables.

ForEach-Item

A lot of people want to know the price per unit easily and quickly. So we created the best application which is called “For each item.” This application can calculate the price per unit of products.

How to use

1. Select the product

2. Set the price and size

3. Press "Calculate"

More about
our application

3

The project was completed by 3 people in only 3 days. In order to make it happen, we used serval tools and design.

Coding

Two principles are mainly used for application product. The MVC (Model, View, Controller) and KISS (Keep It Simple Stupid) principles. These methods helped our understanding of the app's production much easier, faster, and cleaner. MVC model helped us planning and realizing our app. This was because this design pattern gave us an outline of how our ideas should be organized into actual code. Besides, MVC makes other developers easier to understand our code.

KISS principle helped us keep our code as simple as possible. Whenever we have complex codes, we always try to break them into smaller maintainable codes. Therefore, we are able to implement a quick code.

Scheduling

As for our communication tools, we used Slack and Notion. We mainly communicate through Slack. We also shared documents and mock-up files using Google drive and Slack to make our product efficiently.
In terms of the meetings, we had group meetings every day throughout the week. We were able to have meetings regardless of time and location by using the Google Meet service.

We created the to-do lists every day using Notion. Especially, we used the new timeline feature in Notion, which always allowed us to keep track of our progress and relevant conversations.

Landing page

To promote our product. We also create a Landing page (this current page) on Github Pages. We simply used HTML, SCSS, and javascript. The page was created within 5 days including a responsive feature. This simple static pages include not only basic information but also survey result and animation explanation so that users can easily, effectively understand.

Future work

As an improvement for our project, we're mainly planning to implement two big features. One is expanding as a Mobile application. We've created desktop application as a prototype. For realizing more effective and efficient usage, mobile app will be a great opportunity to fulfill users' satisfaction.

The second work is using API as database. By using API as database, we will be able to suggest more flexible and variable options to users.
